# RBAC + матрица прав

## Роли платформы

`user` (соискатель), `vip-user` (привилегии — буст в выдаче, ранний доступ; конкретика в конфиг-флагах), `admin` (модератор платформы).

## Роли внутри компании (team_member.company_role)

`main_manager` (владелец кабинета — полный доступ, права не отключаемы), `manager`, `coordinator`, `staff` — по выданным правам.

Матрица прав (bool-поля team_member): `create` (создавать события/вакансии), `hire` (нанимать/статусы откликов/шортлист/ЧС), `finance` (баланс, пополнение, операции), `invite` (пригласительные ссылки). Выдаёт права только `main_manager`.

## Инвайт-ссылки

Приглашение в команду — ссылка `join/<code>` с ролью, сроком действия (24ч / 7д / 30д), лимитом и счётчиком переходов, возможностью отозвать. Переход = вступление с указанной ролью.

## Правила

- Каждый защищённый эндпоинт — зависимость проверки роли/права (`require_permission(...)`); проверка в одном месте, не размазана по сервисам.
- На каждый запрет — негативный тест (в т.ч.: компания не видит KYC-документы; manager без `finance` не видит баланс; blacklist скрывает соискателя из откликов компании).
- Объектная авторизация обязательна: членство именно в этой компании/филиале, владение именно этой заявкой.
