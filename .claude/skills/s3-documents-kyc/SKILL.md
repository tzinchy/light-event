# Документы, S3 и KYC

- Все файлы (паспорт, селфи с паспортом, медкнижка, диплом, пруфы платежей, аватары) — в хранилище за интерфейсом `Storage` (app/core/storage.py). Основной backend — S3-совместимый (MinIO в dev); **если S3 недоступен при старте (storage_backend=auto) — автоматический фолбэк в локальную папку** (`local_storage_path`, по умолчанию var/storage), папка создаётся сама. В БД — метаданные: `storage_key`, mime, size, статус.
- Хранилище приватное; контент отдаётся только через API-эндпоинт с проверкой прав (работает одинаково для S3 и папки), ключи объектов не публикуются.
- Статусы документа: `missing` (записи нет) / `pending` (на проверке) / `verified` / `rejected` (+причина).
- KYC-модерация — очередь в админке: админ смотрит документ, одобряет/отклоняет; авто-flag (напр. «требует ручной проверки») — подсказка, не решение.
- **Жёсткое ограничение доступа:** содержимое KYC-документов видят только владелец и admin. Компания/работодатель получает ТОЛЬКО статус («Подтверждено»/«На проверке»), никогда — сам документ или presigned URL. На это — обязательные негативные тесты.
- Загрузка: клиент получает presigned upload URL → грузит в S3 → создаётся запись метаданных со статусом `pending`.
