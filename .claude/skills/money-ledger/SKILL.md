# Деньги: копейки + атомарный леджер

- Все суммы — целочисленные копейки (`BIGINT`, суффикс `_kop`). Никаких float/Decimal в хранении; форматирование в рубли — только на выводе.
- Тарифы в конфиге, не в коде: публикация смены 99_000 коп (990 ₽), тест компании 150_000 коп (1 500 ₽), комиссия платформы 6%.
- **Двойная запись:** каждая операция — запись в `ledger_entry` (debit_account → credit_account, amount_kop > 0, kind, ref). Журнал append-only: UPDATE/DELETE по нему запрещены.
- Балансы `account.available_kop / on_hold_kop` — денормализованный кэш поверх журнала; в тестах сверяются пересчётом из журнала.
- **Атомарность:** списание/зачисление/hold — одна транзакция БД с `SELECT ... FOR UPDATE` по затронутым счетам. Недостаток средств → ошибка домена, транзакция откатывается целиком (включая, напр., создание вакансии).
- **Пополнение — только через ручную модерацию:** пользователь/компания создаёт `topup_request` с пруфом платежа (файл в S3), админ подтверждает → атомарное зачисление. Внешнего платёжного провайдера нет; слой изолирован за интерфейсом `PaymentProvider`, чтобы провайдер можно было добавить позже.
- Выплаты соискателям: средства резервируются в `on_hold`, админ «проводит выплату» → списание с on_hold + комиссия платформе, всё в одной транзакции.
